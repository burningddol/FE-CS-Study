# FE-CS-Study

> 비전공으로 FE에 들어왔는데, 면접에서 CS 얘기가 나올 때마다 자꾸 말이 막혔다. 그래서 시작한 스터디.

**시작**: 2026-04-19
**기간**: 10주 (하루 3시간, 주 6일 기준)
**이전 스터디**: [JS-Study](https://github.com/burningddol/JS-Study---) — 언어 자체를 파고들었던 기록

---

## 왜 시작했나

부트캠프에선 React, Next, TS를 충분히 다뤘다. 그런데 "브라우저는 왜 멀티 프로세스예요?", "V8이 정확히 뭘 해요?" 같은 질문 앞에선 늘 뭔가 빠진 것 같았다. 프레임워크를 **쓰는 건** 되는데 **이해하는 건** 아니었던 거다.

전공자가 4년 걸려 배운 걸 10주로 줄일 순 없다. 대신 FE 실무랑 면접에 바로 닿는 것만 추려서 밀도 있게 파고드는 게 목표다.

끝났을 때 이 정도는 스스로 답할 수 있으면 된다.

- 브라우저는 왜 탭마다 프로세스를 쓰는 걸까
- `0.1 + 0.2`가 `0.3`이 아닌 건 비트 수준에서 무슨 일이 벌어져서 그런 걸까
- Babel은 내 코드를 어떻게 "변환"한다고 말하는 걸까
- Vite가 Webpack보다 빠르다길래 믿었는데, 정확히 어디가 빠른 걸까
- OAuth 2.0 흐름을 화이트보드에 막힘없이 그릴 수 있나
- Virtual DOM이 빠른 게 아니라면, 진짜로 빠른 건 뭐였나

## 어떻게 공부할 건가

### 3회독을 깔고 시작한다
한 번에 다 이해하려다 보면 진도도 안 나가고 기분만 나빠진다. 1회독은 "용어에 익숙해지기", 2회독은 "어디에 쓰이는지 연결하기", 3회독은 "면접에서 5분 답하기" — 이렇게 나눠서 간다.

### 아웃풋이 없으면 안 한 거다
챕터 끝날 때마다 Wiki에 **내 말로** 다시 써 본다. 읽은 대로 옮기면 파인만 테크닉이 아니라 받아쓰기다. 면접 질문은 음성으로 녹음하면서 답변해 본다. 입으로 뱉어 보면 어디서 막히는지 바로 티가 나더라.

### 손으로 그릴 수 없으면 이해한 게 아니다
TCP 3-way handshake, OAuth 흐름, 렌더링 파이프라인 — 이런 건 펜 없이도 그릴 수 있어야 넘어간 걸로 친다. AST는 [astexplorer.net](https://astexplorer.net)에 직접 코드 넣어 보는 게 제일 빠르다.

### 코테는 그냥 매일 조금씩
하루 한 문제, 30분 고민 후 해설. 몰아서 하면 안 늘더라. 주말에 한 주 동안 푼 거 돌아보는 정도면 충분하다.

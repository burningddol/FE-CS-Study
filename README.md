# FE-CS-Study

비전공 출신 FE 개발자의 CS 집중 스터디. 10주 동안 전공자 수준의 기초 체력을 올리는 게 목표다.

**시작**: 2026-04-19
**기간**: 10주 (하루 3시간, 주 6일)
**이전 스터디**: [JS-Study](https://github.com/burningddol/JS-Study---) — 언어 기초편

---

## 왜

프레임워크를 쓰는 건 문제없지만, 그 밑에서 뭐가 돌아가는지는 여전히 흐릿하다. 전공자가 4년간 쌓는 걸 10주로 줄일 순 없으니까, FE 실무·면접에 직결되는 것만 추려서 파고든다.

끝났을 때 아래 질문들에 스스로 답할 수 있으면 된다.

- 브라우저는 왜 탭마다 프로세스를 분리하나
- `0.1 + 0.2 !== 0.3`인 이유는 비트 수준에서 무엇인가
- Babel이 코드를 변환한다는 건 정확히 무슨 뜻인가
- Vite가 Webpack보다 빠른 건 어디가 빨라서인가
- OAuth 2.0 흐름을 화이트보드에 그릴 수 있는가
- Virtual DOM이 빠르다고 하는데, 사실 빠른 건 무엇인가

## 공부 방식

- **3회독 전제.** 1회독은 용어, 2회독은 구현·실전 연결, 3회독은 면접 답변 수준.
- **아웃풋 필수.** 챕터 끝나면 wiki에 내 말로 다시 정리.
- **그림으로 검증.** TCP handshake, OAuth 흐름, 렌더링 파이프라인은 펜 없이 그릴 수 있어야 넘어간 걸로 본다.
- **코테는 매일 한 문제.** 30분 고민 후 해설. 몰아서 풀지 않는다.

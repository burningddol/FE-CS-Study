## 왜 이 스터디인가

부트캠프에서 FE 프레임워크 사용법은 익혔지만, 그 아래 **"왜 그렇게 동작하는가"**에 대한 설명은 부족합니다. 전공자가 4년간 배우는 CS 커리큘럼 중 **FE 실무와 면접에 직결되는 것만** 추려 10주에 끝내는 것이 목표입니다.

- 브라우저는 왜 멀티 프로세스인가
- `0.1 + 0.2 !== 0.3`는 어떤 비트 수준의 이유인가
- Babel은 내 코드를 어떻게 변환하는가
- Vite는 Webpack보다 왜 빠른가
- OAuth 2.0 흐름을 화이트보드에 그릴 수 있는가
- Virtual DOM이 빠른 게 아니라 무엇이 빠른가

## 학습 원칙

### 1. 3회독 전제
한 번에 다 이해하려 하지 말 것.
- **1회독**: 용어와 큰 그림
- **2회독**: 구현과 실전 연결
- **3회독**: 면접에서 5분 답변 가능한 수준

### 2. 아웃풋 위주
- 챕터 끝날 때마다 Wiki에 **내 말로** 정리 (파인만 테크닉)
- 면접 질문에 **음성 녹음**하며 답변 연습
- 모든 개념은 "내 프로젝트 어디에 쓰이지?"로 연결

### 3. 손으로 그리기
- TCP 3-way handshake, OAuth 흐름, 렌더링 파이프라인은 **화이트보드에 그려서** 설명 가능해야 함
- AST는 [astexplorer.net](https://astexplorer.net)에서 직접 눈으로 확인

### 4. 코테 병행
매일 1문제, 30분 고민 후 해설. 주말엔 주간 복기.

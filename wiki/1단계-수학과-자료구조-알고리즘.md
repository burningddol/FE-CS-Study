# 1단계: 수학·논리 + 자료구조·알고리즘

전공자와 가장 크게 벌어지는 영역이다. 단순 코테 풀이가 아니라, 알고리즘이 왜 그 복잡도를 갖는지 설명할 수 있어야 한다. 수학·논리 기초를 먼저 잡고, 자료구조·알고리즘 순으로 진행한다.

**기간**: 2.5주

* * *

## 1. 수 표현과 비트 연산

`0.1 + 0.2 !== 0.3`을 비트 수준에서 설명할 수 있어야 한다.

### 1-1. 2진수·16진수·2의 보수
- 10진수 ↔ 2진수 ↔ 16진수 변환
- 음수는 2의 보수로 표현한다. 덧셈 회로 하나로 뺄셈까지 처리된다
- JS 비트 연산은 내부적으로 32비트 signed integer로 변환된다

### 1-2. IEEE 754 부동소수점
- 64비트 double: 부호 1 / 지수 11 / 가수 52
- `0.1`을 2진수로 바꾸면 무한소수이므로 정밀도 손실이 발생한다
- 비교할 때는 `Math.abs(a - b) < Number.EPSILON`을 쓴다
- `Number.MAX_SAFE_INTEGER = 2^53 - 1`. 초과 시 `BigInt`를 써야 한다

### 1-3. 비트 연산자
- `&` `|` `^` `~` `<<` `>>` `>>>`
- 짝/홀 판정: `n & 1`
- 2의 거듭제곱 판정: `(n & (n - 1)) === 0`
- 플래그 집합: `READ | WRITE | EXECUTE`

### 1-4. FE 실전 연결
- 권한 비트마스크 (RBAC)
- 색상 RGB 합성 (`0xRRGGBB`)
- `n << 1`은 `* 2`, `n >> 1`은 `Math.floor(n / 2)`. 부호 있는 정수에서만 안전하다는 점에 주의해야 한다

* * *

## 2. 이산수학 핵심

알고리즘 분석과 조건문 리팩토링의 근거를 알아야 한다.

### 2-1. 명제 논리와 드모르간 법칙
- `!(A && B) === (!A || !B)`
- 복잡한 `if` 조건을 단순화하는 수학적 근거

### 2-2. 집합과 배열 메서드
- 합집합 ↔ `[...a, ...b]`, 교집합 ↔ `a.filter(x => b.includes(x))`
- Set을 쓰면 집합 연산을 O(n)에 수행할 수 있다

### 2-3. 순열·조합
- `nPr`, `nCr`, 중복 순열
- 백트래킹 문제의 수학적 바탕

### 2-4. 모듈러 연산
- 해시 함수가 `% size`를 쓰는 이유
- JS에서 `-1 % 5`는 `-1`이다. 언어별 동작에 주의해야 한다

### 2-5. 수학적 귀납법
- 재귀 정당성 증명의 기초
- 반복문 불변량(invariant)의 뿌리

* * *

## 3. 복잡도 분석

코드를 보고 시간/공간 복잡도를 즉답할 수 있어야 한다.

### 3-1. 빅오·빅오메가·빅세타
- 빅오: 상한 / 빅오메가: 하한 / 빅세타: 상하한 일치
- 실무에서는 빅오만 써도 되지만 정의는 알아야 한다

### 3-2. 대표 복잡도 직관
- O(1): 해시 조회, 배열 인덱싱
- O(log n): 이분 탐색, B-Tree
- O(n log n): 효율 정렬의 하한
- O(n²): 이중 루프
- O(2ⁿ): 부분집합 생성, 나이브 재귀

### 3-3. 분할상환 분석
- `Array.push`가 평균 O(1)인 이유
- 동적 배열의 2배 확장 전략

### 3-4. JS 내부의 숨은 복잡도
- `Array.sort()` = Timsort, 평균 O(n log n)
- `String.includes()` = 단순 O(n·m), KMP 아님
- `Array.indexOf()` = O(n). Set/Map을 쓰면 O(1)

* * *

## 4. 배열·문자열·해시

조회/중복 문제를 보면 해시부터 떠올릴 수 있어야 한다.

### 4-1. 배열 연산 비용
| 연산 | 복잡도 | 비고 |
|---|---|---|
| 인덱싱 `a[i]` | O(1) | 연속 메모리 |
| `push` / `pop` | 평균 O(1) | 끝에서 작업 |
| `unshift` / `shift` | O(n) | 전부 재배치 |
| `splice` | O(n) | 위치에 따라 다름 |
| `slice` | O(k) | k = 자를 길이 |

### 4-2. 문자열 불변성
- JS 문자열은 immutable이다. 연결할 때마다 새 객체가 생성된다
- 긴 문자열을 만들 때는 배열에 push 후 `Array.join`을 써야 한다

### 4-3. 투 포인터 / 슬라이딩 윈도우
- 정렬된 배열에서 합 찾기
- 연속 부분합 최대/최소

### 4-4. 해시 테이블 내부
- 해시 함수 → 버킷 인덱스
- 충돌 해결: 체이닝(JS Map이 가까움) vs 개방 주소법
- 평균 O(1), 최악 O(n)인 이유: 모든 키가 한 버킷에 쏠릴 때

### 4-5. Object vs Map vs Set
- Object: 문자열/심볼 키만 가능, 프로토타입 오염 위험
- Map: 모든 값이 키로 가능, 삽입 순서 유지, 크기 조회 O(1)
- Set: 중복 없는 값 집합

**풀어야 할 문제**: "두 수의 합", "완주하지 못한 선수", "폰켓몬", "의상"

* * *

## 5. 스택·큐·힙

LIFO/FIFO 상황을 구분하고 힙까지 직접 구현해야 한다.

### 5-1. 스택 (LIFO)
- 콜 스택, 브라우저 히스토리, Undo/Redo
- 괄호 검증, 후위 표기식 계산

### 5-2. 큐 (FIFO)
- 이벤트 루프의 태스크 큐 / 마이크로태스크 큐 (2단계에서 심화)
- BFS의 기본

### 5-3. 덱 (Deque)
- 양쪽 삽입/삭제 O(1)
- 슬라이딩 윈도우 최댓값 문제

### 5-4. 힙과 우선순위 큐 — 직접 구현
- 이진 힙: 완전 이진 트리 + 힙 속성
- 삽입 / 추출 모두 O(log n)
- 배열 기반: 부모 `(i-1)/2`, 자식 `2i+1`, `2i+2`
- 다익스트라, A*의 핵심

```js
class MinHeap {
  heap = [];
  push(v) { /* 공부하면서 채우기 */ }
  pop() { /* 공부하면서 채우기 */ }
}
```

## 6. 연결 리스트·트리·그래프

DOM, Virtual DOM, 번들 의존성이 왜 이 구조인지 이해해야 한다.

### 6-1. 연결 리스트
- 단일/이중/원형
- 배열 대비: 삽입/삭제 O(1) vs 랜덤 접근 O(n)
- 뒤집기 패턴 (포인터 3개)

### 6-2. 이진 트리·BST
- 순회: 전위/중위/후위, 레벨(BFS)
- BST 탐색/삽입/삭제 평균 O(log n), 최악 O(n)
- 균형 트리(AVL, Red-Black)는 이름만

### 6-3. 트라이 (Trie)
- 문자열 검색 자동완성 자료구조
- 삽입·검색 O(m), m = 문자열 길이
- 실전: 주소 검색, 태그 필터

### 6-4. Union-Find
- 연결성 판정
- Path Compression + Union by Rank → 거의 O(1)
- Kruskal MST의 핵심

### 6-5. 그래프 표현
- 인접 행렬: O(V²) 공간, 간선 확인 O(1)
- 인접 리스트: O(V + E) 공간. 대부분 이 방식을 쓴다

### 6-6. DFS / BFS
- DFS: 재귀 또는 스택. 백트래킹·위상 정렬·SCC
- BFS: 큐. 가중치 없는 최단 경로

**FE 연결**: DOM 트리, Virtual DOM diff, 컴포넌트 트리, 번들러 의존성 그래프.

## 7. 핵심 알고리즘

문제 유형을 보면 어떤 알고리즘을 쓸지 즉답할 수 있어야 한다.

### 7-1. 정렬
- 버블/선택/삽입: 개념만
- 퀵: 평균 O(n log n), 최악 O(n²), in-place
- 머지: O(n log n) 보장, stable, 추가 공간
- 힙: O(n log n), in-place, unstable
- Timsort: JS `Array.sort()`의 내부

### 7-2. 이분 탐색
- 정렬된 배열에서 O(log n)
- `lower_bound` / `upper_bound`
- 파라메트릭 서치: 답을 이분 탐색

### 7-3. 재귀와 반복
- 꼬리 재귀 최적화는 JS 엔진 대부분 미지원
- 재귀 깊이 주의

### 7-4. 백트래킹
- N-Queens, 순열/조합, 스도쿠
- 가지치기(pruning)가 핵심

### 7-5. 분할 정복
- 머지·퀵 정렬, Strassen 행렬 곱셈

### 7-6. 동적 계획법 (DP)
- Top-down(메모이제이션) vs Bottom-up(타뷸레이션)
- 피보나치, LIS, LCS, 배낭, 동전 교환
- 최적 부분 구조 + 중복 부분 문제

### 7-7. 그리디
- 정당성 증명이 생명. DP로 풀리는 걸 그리디로 착각하지 않아야 한다

### 7-8. 위상 정렬
- DAG에서 선후 관계 결정
- 번들러가 모듈 순서를 정하는 원리
- Kahn(진입 차수) / DFS 후위 순회

### 7-9. 최단 경로
- 다익스트라: 음수 가중치 없음, O((V + E) log V)
- 벨만-포드: 음수 허용, O(VE)
- 플로이드-워셜: 모든 쌍, O(V³)

### 7-10. 편집 거리 (Levenshtein)
- DP의 대표 문제
- diff, 검색 유사도, 오타 제안

## 연습 문제

### 기본
1. `0.1 + 0.2`를 16자리까지 출력하고, IEEE 754 비트로 이유를 설명하라
2. 정수 n이 2의 거듭제곱인지 비트 연산으로 판정하는 함수 작성
3. 배열 중복 제거를 Object / Map / Set으로 각각 구현하고 복잡도 비교

### 심화
4. MinHeap을 배열로 구현하고 Top-K 문제에 적용
5. 문자열 배열을 트라이에 넣고 접두사 검색 함수 작성
6. 위상 정렬 구현 후 모듈 의존성 순서에 적용
7. 두 문자열의 편집 거리를 DP로 구하라

### FE 연결
8. 디바운스 함수 구현, 태스크 큐를 건드리는 이유 설명

# 1단계: 수학·논리 + 자료구조·알고리즘

> 전공자와 가장 많이 벌어지는 영역. **수학·논리 기초**를 먼저 잡고 자료구조/알고리즘으로 넘어갑니다. 단순 코테 풀이가 아니라, 알고리즘이 **왜** 그 복잡도를 갖는지 증명할 수 있는 수준을 목표로 합니다.

**학습 기간**: 2.5주

* * *

## 1. 수 표현과 비트 연산

컴퓨터가 숫자를 저장하는 실제 방식. **`0.1 + 0.2 !== 0.3`을 내부 비트 수준에서 설명할 수 있으면 통과.**

### 1-1. 2진수·16진수·2의 보수
- 10진수 ↔ 2진수 ↔ 16진수 변환
- 음수 표현: 2의 보수가 왜 부호-크기 방식보다 유리한가
- JS의 정수 연산은 32비트 signed integer로 내부 변환된다

### 1-2. IEEE 754 부동소수점
- 64비트 double: 부호 1 / 지수 11 / 가수 52
- `0.1`을 2진수로 표현하면 무한 반복 → 정밀도 손실
- `Number.EPSILON`으로 비교하는 이유
- `Number.MAX_SAFE_INTEGER = 2^53 - 1`, 초과 시 `BigInt`

### 1-3. 비트 연산자
- `&` `|` `^` `~` `<<` `>>` `>>>`
- 짝/홀 판정: `n & 1`
- 2의 거듭제곱 판정: `(n & (n - 1)) === 0`
- 플래그 집합: `READ | WRITE | EXECUTE`

### 1-4. FE 실전 연결
- 권한 비트마스크 (RBAC)
- 색상 RGB 합성 (`0xRRGGBB`)
- 성능 트릭: `n << 1`은 `n * 2`, `n >> 1`은 `Math.floor(n / 2)` (단, 부호 있는 정수만)

* * *

## 2. 이산수학 핵심

### 2-1. 명제 논리와 드모르간 법칙
- `!(A && B) === (!A || !B)`
- 복잡한 `if` 조건을 리팩토링하는 수학적 근거

### 2-2. 집합과 배열 메서드
- 합집합 ↔ `[...a, ...b]`, 교집합 ↔ `a.filter(x => b.includes(x))`
- Set을 쓰면 집합 연산이 O(n)에 가능한 이유

### 2-3. 순열·조합
- `nPr`, `nCr`, 중복 순열
- 백트래킹 문제의 수학적 베이스

### 2-4. 모듈러 연산
- 해시 함수가 `% size`를 쓰는 이유
- 음수 모듈러 주의 (`-1 % 5`는 언어마다 다름, JS는 `-1`)

### 2-5. 수학적 귀납법
- 재귀 정당성 증명의 기초
- 반복문 불변량(invariant)과의 연결

* * *

## 3. 복잡도 분석

### 3-1. 빅오·빅오메가·빅세타
- 빅오: 상한, 빅오메가: 하한, 빅세타: 상하한 일치
- 일반적으로 빅오만 말해도 되지만 정의는 알아야 함

### 3-2. 대표 복잡도 직관
- O(1): 해시 조회, 배열 인덱싱
- O(log n): 이분 탐색, B-Tree
- O(n log n): 효율 정렬의 하한
- O(n²): 이중 루프
- O(2ⁿ): 부분집합 생성, 나이브 재귀

### 3-3. 분할상환 분석
- `Array.push`가 왜 "평균 O(1)"인가
- 동적 배열의 2배 확장 전략

### 3-4. JS 내부의 숨은 복잡도
- `Array.sort()` = Timsort, 평균 O(n log n)
- `String.includes()` = 단순 O(n·m), KMP 아님
- `Array.indexOf()` = O(n), Set/Map 쓰면 O(1)

* * *

## 4. 배열·문자열·해시

### 4-1. 배열 연산 비용
| 연산 | 복잡도 | 비고 |
|---|---|---|
| 인덱싱 `a[i]` | O(1) | 연속 메모리 |
| `push` / `pop` | 평균 O(1) | 끝에서 작업 |
| `unshift` / `shift` | O(n) | 전부 재배치 |
| `splice` | O(n) | 위치에 따라 다름 |
| `slice` | O(k) | k = 자를 길이 |

### 4-2. 문자열 불변성
- JS 문자열은 immutable → 연결할 때마다 새 객체 생성
- 긴 문자열 빌드는 `Array.join` 사용

### 4-3. 투 포인터 / 슬라이딩 윈도우
- 정렬된 배열에서 합 찾기
- 연속 부분합 최대/최소

### 4-4. 해시 테이블 내부
- 해시 함수 → 버킷 인덱스
- 충돌 해결: 체이닝 (JS의 Map이 이 방식에 가까움) vs 개방 주소법
- 평균 O(1), 최악 O(n)인 이유: 모든 키가 같은 버킷에 쏠리는 경우

### 4-5. Object vs Map vs Set
- Object: 문자열/심볼 키, 프로토타입 오염 위험
- Map: 모든 값 키 가능, 삽입 순서 유지, 크기 조회 O(1)
- Set: 중복 없는 값 집합

**실전 문제**: "두 수의 합", "완주하지 못한 선수", "폰켓몬", "의상"

* * *

## 5. 스택·큐·힙

### 5-1. 스택 (LIFO)
- 콜 스택, 브라우저 히스토리, Undo/Redo
- 괄호 검증, 후위 표기식 계산

### 5-2. 큐 (FIFO)
- 이벤트 루프의 **태스크 큐 / 마이크로태스크 큐** (2단계에서 심화)
- BFS 구현의 기본

### 5-3. 덱 (Deque)
- 양쪽 삽입/삭제 O(1)
- 슬라이딩 윈도우 최댓값 문제

### 5-4. 힙과 우선순위 큐 (★ 직접 구현)
- 이진 힙: 완전 이진 트리 + 힙 속성
- 삽입 O(log n), 최솟값/최댓값 추출 O(log n)
- 배열 기반 구현: 부모 `(i-1)/2`, 자식 `2i+1`, `2i+2`
- 다익스트라, A* 알고리즘의 핵심 자료구조

```js
class MinHeap {
  heap = [];
  push(v) { /* 내용 작성 */ }
  pop() { /* 내용 작성 */ }
}
```

## 6. 연결 리스트·트리·그래프

### 6-1. 연결 리스트
- 단일/이중/원형
- 배열 대비: 삽입/삭제 O(1) vs 랜덤 접근 O(n)
- 뒤집기 패턴 (포인터 3개)

### 6-2. 이진 트리·BST
- 순회: 전위/중위/후위, 레벨 (BFS)
- BST 탐색/삽입/삭제 평균 O(log n), 최악 O(n)
- 균형 트리 (AVL, Red-Black) — 이름만

### 6-3. 트라이 (Trie)
- 문자열 검색 자동완성 구조
- 삽입·검색 O(m), m = 문자열 길이
- 실전: 주소 검색, 태그 필터

### 6-4. Union-Find
- 연결성 판정 자료구조
- Path Compression + Union by Rank → 거의 O(1)
- MST(Kruskal)의 핵심

### 6-5. 그래프 표현
- 인접 행렬: O(V²) 공간, 간선 확인 O(1)
- 인접 리스트: O(V + E) 공간, 대부분 이걸 씀

### 6-6. DFS / BFS
- DFS: 재귀 또는 스택, 백트래킹·위상 정렬·SCC
- BFS: 큐, 최단 경로 (가중치 없음)

**FE 연결**: DOM 트리, Virtual DOM diff, 컴포넌트 트리, **번들러 의존성 그래프**

* * *

## 7. 핵심 알고리즘

### 7-1. 정렬
- 버블/선택/삽입 — O(n²), 개념만
- 퀵 정렬 — 평균 O(n log n), 최악 O(n²), **in-place**
- 머지 정렬 — O(n log n) 보장, stable, 추가 공간
- 힙 정렬 — O(n log n), in-place, unstable
- **Timsort** — JS `Array.sort()`, 머지 + 삽입의 하이브리드

### 7-2. 이분 탐색
- 정렬된 배열에서 O(log n)
- `lower_bound` / `upper_bound` 개념
- 파라메트릭 서치 (답을 이분 탐색)

### 7-3. 재귀와 반복
- 꼬리 재귀 최적화 (JS 엔진 대부분 미지원)
- 재귀 깊이 제한 주의

### 7-4. 백트래킹
- N-Queens, 순열/조합 생성, 스도쿠
- 가지치기(pruning)의 중요성

### 7-5. 분할 정복
- 머지 정렬, 퀵 정렬, Strassen 행렬 곱셈

### 7-6. 동적 계획법 (DP)
- 메모이제이션 (Top-down) vs 타뷸레이션 (Bottom-up)
- 피보나치, LIS, LCS, 배낭, 동전 교환
- 최적 부분 구조 + 중복 부분 문제

### 7-7. 그리디
- 정당성 증명이 핵심 — **DP로 풀리는데 그리디처럼 보이는 문제** 주의

### 7-8. 위상 정렬
- DAG에서 선후 관계 결정
- **번들러 의존성 해결 원리**
- Kahn 알고리즘 (진입 차수 기반) / DFS 후위 순회

### 7-9. 최단 경로
- 다익스트라: 음수 가중치 없음, O((V + E) log V)
- 벨만-포드: 음수 허용, O(VE)
- 플로이드-워셜: 모든 쌍, O(V³)

### 7-10. 편집 거리 (Levenshtein)
- DP 문제의 대표
- diff, 검색 유사도, 오타 제안

## 연습 문제

### 기본 (복습용)
1. `0.1 + 0.2`의 실제 결과를 16자리까지 출력하고, IEEE 754 비트로 왜 그런지 설명하라.
2. 정수 n이 2의 거듭제곱인지 판정하는 함수를 비트 연산으로 작성하라.
3. 배열에서 중복 제거를 Object / Map / Set 세 방법으로 구현하고 복잡도를 비교하라.

### 심화
4. MinHeap을 배열로 직접 구현하고 Top-K 문제를 풀어라.
5. 문자열 배열을 트라이에 넣고 접두사 검색 함수를 작성하라.
6. 위상 정렬을 구현하고, 모듈 의존성 순서 결정에 적용하라.
7. 두 문자열의 편집 거리를 DP로 구하라.

### FE 연결
8. 디바운스 함수를 구현하고 그 동작이 왜 "태스크 큐"를 건드리는지 설명하라.
9. Virtual DOM diff가 왜 편집 거리의 단순화 버전인지 설명하라.

## 체크리스트

- [ ] 부동소수점 오차의 원인을 비트 수준에서 설명할 수 있다
- [ ] `& | ^ << >> >>>` 각각의 용도를 즉답할 수 있다
- [ ] 드모르간 법칙으로 복잡한 `if` 조건을 리팩토링할 수 있다
- [ ] 빅오를 상한의 수학적 정의로 말할 수 있다
- [ ] `Array.sort()`가 내부에서 무엇을 쓰는지 안다
- [ ] 해시 테이블이 최악 O(n)인 조건을 안다
- [ ] MinHeap을 배열로 구현할 수 있다
- [ ] BFS와 DFS의 구현 차이를 스택/큐로 설명할 수 있다
- [ ] 트라이의 시간/공간 복잡도를 안다
- [ ] Union-Find의 최적화 기법 2개를 말할 수 있다
- [ ] 위상 정렬을 Kahn 알고리즘으로 구현할 수 있다
- [ ] 다익스트라가 왜 우선순위 큐를 쓰는지 설명할 수 있다
- [ ] DP와 그리디의 판단 기준을 안다
- [ ] Timsort가 Merge/Insertion의 하이브리드인 이유를 안다
- [ ] 편집 거리 DP를 칠판에 그릴 수 있다
